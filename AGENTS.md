# AGENTS.md

## Назначение

Этот файл задаёт правила для агентской работы в репозитории `SDG`.

## Язык

- Весь пользовательский текст, документация, названия экранов, описания API и комментарии в новых файлах писать на русском языке, если задача явно не требует иного.
- Английский допускается только там, где это продиктовано стеком, кодом, именами библиотек, переменными окружения, HTTP-маршрутами и типами.

## Среда

- Рабочая среда: Windows.
- Основная оболочка: PowerShell.
- Для поиска по проекту использовать PowerShell-команды, а не `rg`.

## Принципы работы

- Сначала понять задачу, потом менять код.
- Если есть неоднозначность, назвать её явно, а не выбирать молча.
- Предпочитать простое решение минимального объёма.
- Вносить только целевые изменения без побочных рефакторингов.
- Не улучшать соседний код, если это не требуется задачей.
- Не добавлять абстракции на будущее, если в задаче этого нет.

## Правила изменений

- Каждый изменённый файл должен быть напрямую связан с задачей.
- Новые файлы и структура должны соответствовать уже принятому стилю проекта.
- Не удалять чужой код без явного запроса.
- Если изменение делает мои же импорты, переменные или функции неиспользуемыми, убирать их.

## Документация

- Крупные ТЗ, архитектурные решения и проектные правила выносить в отдельные `.md`-файлы.
- Не хранить объёмные требования только в переписке.
- Для новых документов использовать понятные имена файлов и предсказуемое расположение, обычно в `docs/`.

## Реализация

- Для frontend использовать SCSS Modules, а не общие огромные стили.
- Бизнес-логику не смешивать с UI.
- Игровые правила держать отдельно от визуального слоя.
- `game-core` должен оставаться чистым TypeScript-модулем без зависимостей от React, Phaser, Socket.IO, Fastify, Supabase и DOM API.

## Проверка результата

Перед завершением задачи нужно по возможности проверить:

- что созданные файлы лежат в ожидаемых местах;
- что названия, структура и формулировки соответствуют задаче;
- что изменения минимальны и не затрагивают лишнее.
